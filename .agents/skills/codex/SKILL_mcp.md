# Codex タスク委任スキル

Codex（OpenAI のコーディングエージェント）にタスクを委任するスキル。

## 引数

```
/codex [タスクの説明]
```

- **タスクの説明**: Codex に実行させたいタスクの内容（必須）

## 手順

### 1. タスク内容の確認

引数からタスクの説明を取得する。引数が空の場合は AskUserQuestion でタスク内容を確認する。

### 2. Codex セッションの開始

`mcp__codex__codex` を以下のパラメータで呼び出す:

```json
{
  "prompt": "{タスクの説明}",
  "model": "gpt-5.3-codex",
  "cwd": "{プロジェクトルート}",
  "sandbox": "workspace-write",
  "approval-policy": "on-failure",
  "config": {
    "reasoning_effort": "xhigh"
  }
}
```

**パラメータの説明**:
- `model`: GPT-5.3 Codex（reasoning effort: xhigh）
- `cwd`: 現在のプロジェクトルートディレクトリ
- `sandbox`: プロジェクトディレクトリへの書き込みのみ許可
- `approval-policy`: コマンド失敗時のみ承認を要求

### 3. 結果の確認

Codex の実行結果をユーザーに報告する:
- 成功した場合: 変更内容のサマリーを提示
- 失敗した場合: エラー内容を報告し、次のアクションを提案

### 4. フォローアップ（必要に応じて）

結果に対してユーザーから追加指示があった場合、`mcp__codex__codex-reply` で同一スレッドを継続する:

```json
{
  "threadId": "{前回のスレッドID}",
  "prompt": "{追加の指示}"
}
```

## 注意事項

- ユーザーの依頼内容を変えずにそのまま Codex に委任すること。事前に情報を収集・加工して prompt に含めない
  - 例: レビュー依頼の場合、差分の取得も Codex 自身に行わせる（呼び出し側で `git diff` して結果を渡さない）
  - 例: バグ調査の場合、関連ファイルの読み取りも Codex に任せる（呼び出し側で事前にファイルを読まない）
- Codex の変更内容は `git diff` で確認し、意図しない変更がないかレビューする
- 大規模な変更を依頼する場合は、タスクを分割して段階的に実行する
